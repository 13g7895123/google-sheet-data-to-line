# 前端畫面規格說明

## 1. 整體佈局（Layout）

系統採用左側導覽列 + 右側主內容區的經典管理後台佈局。

```
┌──────────────────────────────────────────────────┐
│  Header（頂部欄）                                 │
│  ┌─────┬─────────────────────────────────────────┤
│  │     │                                         │
│  │ 側  │          主內容區域                      │
│  │ 邊  │          (Content Area)                  │
│  │ 導  │                                         │
│  │ 覽  │                                         │
│  │ 列  │                                         │
│  │     │                                         │
│  └─────┴─────────────────────────────────────────┘
```

### 1.1 共用元件

| 元件 | 檔案路徑 | 說明 |
|------|---------|------|
| `MainLayout` | `components/layout/MainLayout.tsx` | 最外層佈局容器，包含 Sidebar + Header + Content |
| `Sidebar` | `components/layout/Sidebar.tsx` | 左側導覽列，含 logo、導覽選單、收合按鈕 |
| `Header` | `components/layout/Header.tsx` | 頂部欄，顯示目前頁面標題與使用者資訊 |

### 1.2 Sidebar 導覽選單項目

| 圖示 | 名稱 | 路徑 | 說明 |
|------|------|------|------|
| 📊 | 儀表板 | `/` | Dashboard 總覽 |
| 📋 | Case 管理 | `/cases` | 所有 Case 列表 |
| 📄 | Google Sheets | `/google-sheets` | Sheet 資料來源管理 |
| 💬 | LINE 好友 | `/line-friends` | 好友名單管理 |
| ⏰ | 排程管理 | `/schedules` | 所有排程總覽 |
| 📝 | 發送紀錄 | `/logs` | 歷史推播紀錄 |
| ⚙️ | 系統設定 | `/settings` | API Key 與系統設定 |

### 2.1 儀表板 Dashboard（`/`）

首頁總覽，讓使用者快速掌握系統狀態。

#### 畫面結構

```
┌─────────────────────────────────────────────────┐
│  儀表板                                         │
├─────────┬──────────┬──────────┬─────────────────┤
│ 統計卡片 │ 統計卡片  │ 統計卡片  │ 統計卡片        │
│ Case 總數│ 今日發送數│ 成功率    │ 排程中 Case     │
├─────────┴──────────┴──────────┴─────────────────┤
│                                                  │
│  最近發送紀錄（表格，顯示最近 10 筆）              │
│  ┌──────┬───────┬────────┬────────┬────────┐     │
│  │ 時間  │ Case  │ 接收者  │ 狀態   │ 訊息   │     │
│  ├──────┼───────┼────────┼────────┼────────┤     │
│  │ ...  │ ...   │ ...    │ ✅/❌  │ ...    │     │
│  └──────┴───────┴────────┴────────┴────────┘     │
│                                                  │
│  即將執行的排程（列出最近 5 個待執行排程）          │
│                                                  │
└──────────────────────────────────────────────────┘
```

#### 功能與元件

| 元件 | 說明 |
|------|------|
| `StatCard` | 統計數字卡片，顯示圖示 + 數值 + 標籤 + 趨勢箭頭 |
| `RecentLogsTable` | 最近發送紀錄表格，精簡版 |
| `UpcomingSchedules` | 即將執行的排程列表 |

#### 資料來源

- `GET /api/dashboard/stats` — 統計數據
- `GET /api/logs?limit=10` — 最近紀錄
- `GET /api/schedules?upcoming=true&limit=5` — 即將執行排程

```
│                                                  │
│  分頁：< 1 2 3 ... >                             │
```

#### 功能與元件

| 元件 | 說明 |
|------|------|
| `CaseCard` | Case 卡片，顯示名稱、狀態標籤、資料來源摘要、接收者數量、排程資訊 |
| `CaseList` | Case 列表容器，支援 Grid / List 切換 |
| `StatusFilter` | 狀態篩選下拉（全部 / 草稿 / 啟用 / 暫停 / 完成） |

#### CaseCard 快捷操作按鈕

| 按鈕 | 功能 | 說明 |
|------|------|------|
| 發送 | 立即發送 | 觸發 `POST /api/cases/:id/send`，彈出確認對話框 |
| 編輯 | 進入編輯 | 導航至 `/cases/:id/edit` |
| 紀錄 | 查看歷史 | 導航至 Case 詳情的發送紀錄 tab |
| 刪除 | 刪除 Case | 彈出確認對話框後刪除 |

### 2.3 新增 Case — 步驟式表單（`/cases/new`）

以 Stepper（步驟導引）引導使用者依序設定 Case 的所有資訊。

#### 步驟總覽

```
[Step 1] ──→ [Step 2] ──→ [Step 3] ──→ [Step 4] ──→ [Step 5] ──→ [Step 6]
 基本資訊     資料來源     訊息模板     選擇接收者    排程設定     確認建立
```

---

#### Step 1：基本資訊

```
┌─────────────────────────────────────────────────┐
│  新增 Case — Step 1/6 基本資訊                    │
│  ● ─ ○ ─ ○ ─ ○ ─ ○ ─ ○                         │
├─────────────────────────────────────────────────┤
│                                                  │
│  Case 名稱 *                                     │
│  ┌─────────────────────────────────────────┐     │
│  │ 例如：2026年4月業績通知                    │     │
│  └─────────────────────────────────────────┘     │
│                                                  │
│  描述（選填）                                     │
│  ┌─────────────────────────────────────────┐     │
│  │                                         │     │
│  │                                         │     │
│  └─────────────────────────────────────────┘     │
│                                                  │
│                              [取消]  [下一步 →]   │
└──────────────────────────────────────────────────┘
```

| 欄位 | 必填 | 類型 | 驗證 |
|------|------|------|------|
| Case 名稱 | ✅ | 文字輸入 | 不可為空，最長 100 字 |
| 描述 | ❌ | 多行文字 | 最長 500 字 |

**`SheetSelector` 元件功能：**

| 功能 | 說明 |
|------|------|
| Spreadsheet 下拉 | 從已註冊的 Google Sheet 中選擇，含搜尋功能 |
| 新增 Sheet 按鈕 | 彈出 Modal，輸入 Spreadsheet URL 後自動解析並註冊 |
| Tab 聯動下拉 | 選擇 Spreadsheet 後自動載入該文件的所有工作表 |
| 資料預覽 | 選擇 Tab 後自動顯示前 5 筆資料的表格預覽 |
| 欄位識別 | 自動讀取第一列作為欄位名稱，供後續模板使用 |

**`SheetPreview` 元件功能：**

| 功能 | 說明 |
|------|------|
| 表格呈現 | 以表格形式顯示 Sheet 資料 |
| 筆數控制 | 預設顯示前 5 筆，可切換為 10 筆 |
| 欄位標示 | 高亮顯示可用於模板的欄位名稱 |

### 2.4 Case 詳情頁（`/cases/:id`）

顯示單一 Case 的完整資訊與操作。

#### 畫面結構

```
┌─────────────────────────────────────────────────┐
│  ← 返回列表    Case：2026年4月業績通知            │
│                 狀態：🟢 ACTIVE                   │
```

#### Tab 內容

|-----|---------|
